Introduction to HTML

What is HTML ?

HTML (Hypertext Markup Language) is a standard markup language used to create web pages. It provides a means to describe the structure and layout of a document, including headings, paragraphs, images, and links. HTML is used in conjunction with other technologies such as CSS and JavaScript to create dynamic and interactive web pages.

The History of HTML

HTML was first developed in the late 1980s by Tim Berners-Lee, a British computer scientist who was working at CERN (the European Organization for Nuclear Research) in Switzerland. Berners-Lee created HTML as a way to structure and share scientific documents over the internet. He also developed the first web browser, called WorldWideWeb (later renamed to Nexus) and the first web server, called CERN httpd.

The first version of HTML, HTML 1.0, was released in 1991. It was a very basic markup language, and only had a handful of tags such as <p> for paragraphs and <a> for links. In 1995, HTML 2.0 was released which added support for tables, images, and forms.

HTML 3.2 was released in 1997, which added support for more advanced features such as style sheets, scripting, and multimedia. HTML 4.0 was released in 1998, which added even more features such as support for internationalisation and accessibility.

HTML5, the current version of HTML, was first released in 2008. It introduced many new features such as semantic elements, multimedia support, and offline storage. HTML5 also introduced a simpler, more flexible syntax, and made it easier to create web applications. HTML is continuously being developed by W3C and WHATWG, adding new tags and functionalities, currently it's on HTML 5.2 and HTML5.3 is under development.

The Purpose of HTML

The primary purpose of HTML is to provide a way to create structured, well-formed documents that can be displayed in web browsers. HTML allows developers to create headings, paragraphs, lists, images, links, and other types of content that can be rendered by a browser to create a webpage.

HTML is used to describe the structure of a webpage, not its presentation or style. For this purpose developers use CSS (Cascading Style Sheets) which is a language used to describe the presentation of an HTML document.

HTML also enables creating forms, which allows users to interact with a website by submitting data. This data can then be processed by the server-side script or JavaScript and displayed on the webpage.

Additionally, HTML can be used to create interactive elements and dynamic content with the help of JavaScript, which is a programming language that allows developers to create complex interactions and animations on a webpage.

In summary, HTML provides a way for developers to create the structure and content of a webpage, CSS provides a way to control the presentation of the webpage, and JavaScript provides a way to make the webpage interactive and dynamic.

How does HTML work?

HTML works by using a set of predefined tags, which are enclosed in angle brackets (< >). These tags are used to describe the structure and layout of a document, and provide information to the web browser about how to display the content.

When a web browser receives an HTML document, it reads the document and interprets the tags to determine how the content should be rendered on the screen.

For example, a <p> tag indicates that the text inside the tag should be displayed as a paragraph, while an <img> tag tells the browser to display an image. A <a> tag indicates a link to another webpage, and the browser will display the linked text as a hyperlink.

The browser reads the HTML document from top to bottom and applies the rules defined by the tags to render the page.

In addition to the standard HTML tags, developers can also create custom tags, called "custom elements", which can be used to create reusable components that can be easily embedded in different pages.

HTML documents are typically stored as plain text files with the .html or .htm file extension, and are served by a web server to a web browser when a user requests the webpage. The web browser then reads the HTML code and renders the webpage accordingly.

The final result should be a webpage that is correctly structured, easy to understand and visually appealing for the user to interact with.

HTML Versions and Standards

HTML has gone through several versions since its initial release in 1991, with each version introducing new features and capabilities. The most recent version is HTML5, which was first released in 2008. The following is a brief overview of the main versions of HTML:

  • HTML 1.0: The first version of HTML, released in 1991. It was a very basic markup language, and only had a handful of tags such as <p> for paragraphs and <a> for links.
  • HTML 2.0: Released in 1995, it added support for tables, images, and forms.
  • HTML 3.2: Released in 1997, it added support for more advanced features such as style sheets, scripting, and multimedia.
  • HTML 4.0: Released in 1998, it added even more features such as support for internationalization and accessibility.
  • HTML5: The current version of HTML, first released in 2008. It introduced many new features such as semantic elements, multimedia support, and offline storage. HTML5 also introduced a simpler, more flexible syntax and made it easier to create web applications.

HTML standards are set by the World Wide Web Consortium (W3C) and are periodically updated to reflect the latest developments in web technology. The W3C is an international organization that sets standards for the web, and their recommendations are widely followed by web developers. It's worth mentioning that there's also an ongoing development of HTML by WHATWG (Web Hypertext Application Technology Working Group), which is an organization that focuses on the development of the web platform. It's important for web developers to keep in mind that not all browsers support the latest version of HTML, so it's important to ensure that the code is compatible with older browsers.

Basic HTML Document Structure

Basic HTML Document Structure: A basic HTML document has a specific structure that must be followed in order for the document to be properly rendered by a web browser. Here is an example of a basic HTML document structure: 

                                                                    
<!DOCTYPE html>
<html>
<head>
<title>My Webpage</title>
</head>
<body>
<h1>Welcome to my webpage</h1>
<p>This is a simple webpage created using HTML.</p>
</body>
</html>
  • The <!DOCTYPE> declaration is used to specify the version of HTML being used. In this case, the <!DOCTYPE html> declaration is used to specify that the document is written in HTML5.
  • The <html> element is the root element of the document and contains all other elements.
  • The <head> element contains information about the document, such as the title of the page, which is displayed in the browser's title bar or tab.
  • The <title> element specifies the title of the document, which is displayed in the browser's title bar or tab.
  • The <body> element contains the content of the document, such as headings, paragraphs, images, and links.
  • The <h1> element is used to create a heading, in this case the title of the webpage.
  • The <p> element is used to create a paragraph.

The closing tags are optional for some HTML tags, but it's a good practice to always close them. This is a basic example and you can add many more tags and attributes to the structure to have a complex webpage, but this is the basic structure that all HTML documents must follow. It's also important to note that HTML5 introduced new semantic tags like <header>, <nav>, <main>, <article>, <aside>, <footer>, among others, that give more meaning to the structure of the webpage, making it more accessible and search engine friendly.

The DOCTYPE Declaration

The DOCTYPE declaration is an instruction that tells the web browser which version of HTML or XML the page is written in. It is the first line of code in an HTML or XML document, and it is required for the document to be properly rendered by the browser.

The DOCTYPE declaration starts with the characters <!DOCTYPE, followed by the name of the document type definition (DTD) and then the name of the root element of the document, for example

                                                                    
                                                                        <!DOCTYPE html>

This is the DOCTYPE declaration for an HTML5 document. It tells the browser that the document is written in HTML5, which is the most recent version of HTML.

In HTML 4 and previous versions, the DOCTYPE declaration was more complex and verbose, it was required to specify the location of the DTD, for example:

                                                                        
                                                                        <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">

The DOCTYPE declaration is important because it tells the browser how to handle the document. If the DOCTYPE declaration is missing or incorrect, the browser may not be able to properly display the page, or may display it in quirks mode, which means that the browser will use a different set of rules to display the page, and it may not look or behave as intended.

It's important to note that the DOCTYPE declaration is case-insensitive, it can be written in uppercase or lowercase.
